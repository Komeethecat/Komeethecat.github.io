속성

1. 속성(Attribute)의 개념

  • 업무에서 필요로 한다.
  • 의미상 더 이상 분리되지 않는다.
  • 엔티티를 설명하고 인스턴스의 구성요소가 된다.

2. 엔터티, 인스턴스와 속성, 속성값에 대한 내용과 표기법

가. 엔터티, 인스턴스, 속성, 속성값의 관계

  • 한 개의 엔터티는 두 개 이상의 인스턴스의 집합이어야 한다.
  • 한 개의 엔터티는 두 개 이상의 속성을 갖는다.
  • 한 개의 속성은 한 개의 속성값을 갖는다.

나. 속성의 표기법

3. 속성의 특징

  • 엔터티와 마찬가지로 반드시 해당 업무에서 필요하고 관리하고자 하는 정보여야 한다.
  • 주식별자에 함수적 종속성을 가져야 한다.
  • 하나의 속성은 한 개의 값만을 가진다. 다중값일 경우 별도의 엔터티를 이용하여 분리한다.

4. 속성의 분류

가. 속성의 특성에 따른 분류

  1. 기본속성 (Basic Attribute)
    • 업무로부터 추출한 모든 속성
    • ex) 제품이름, 제조년월, 제조원가
  2. 설계속성 (Designed Attribute)
    • 원래 업무상 존재하지는 않지만 설계를 하면서 도출해내는 속성
    • 코드성 속성 - 원래 속성을 업무상 필요에 의해 변형
    • 일련번호 - 단일(Unique)한 식별자를 부여하기 위해 모델 상에서 새로 정의
    • ex) 주문번호, 자재번호
  3. 파생속성 (Derived Attribute)
    • 다른 속성으로부터 계산이나 변형이 되어 생성
    • 통계관련 엔터티나 배치 작업이 수행되면서 발생되는 엔터티의 경우 많이 이용됨
    • ex) 순매출액, 위수탁수수료

나. 엔터티 구성방식에 따른 분류

  • PK(Primary Key) 속성 : 엔터티를 식별할 수 있는 속성
  • FK(Foreign Key) 속성 : 다른 엔터티와의 관계에서 포함된 속성
  • 일반속성 : 엔터티에 포함되어 있고 PK, FK에 포함되지 않은 속성
  • 세부 의미 분리 가능 여부
    • 복합 속성 (Composite Attribute) : 주소
    • 단순 속성 (Simple Attribute) : 나이, 성별
  • 속성값의 개수가 2개 이상
    • 다중값 속성 (Multi-Valued Attribute) : 1차 정규화 하거나 별도의 엔터티를 만들어 관계로 연결

5. 도메인 (Domain)

  • 엔터티 내에서 속성에 대한 데이터타입, 크기, 제약사항을 지정하는 것
  • 가질 수 있는 값의 범위

6. 속성의 명명 (Naming)

  1. 해당업무에서 사용하는 이름 이용
  2. 서술식 속성명은 사용하지 않음 (명사형 사용) (소유격도 안됨)
  3. 약어사용 가급적 제한
  4. 전체 데이터모델에서 유일성을 확보하는 것이 좋다.
    • 데이터에 대한 흐름을 파악하고 데이터와 정합성을 유지하는데 큰 도움이 된다.
    • 반정규화를 적용할 때 속성명의 충돌을 방지하여 반정규화를 적용할 수 있게 된다.
